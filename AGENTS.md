# AGENTS.md

1. 이 프로젝트는 고급 의류 쇼핑몰 MVP다.
2. 기본 스택은 Next.js App Router, TypeScript, Tailwind CSS, Supabase다.
3. 변경 전 항상 현재 파일 구조와 기존 패턴을 먼저 확인한다.
4. 사용자 요청이 한국어면 답변과 진행 보고도 한국어로 한다.
5. 민감한 키는 절대 커밋하지 않는다.
6. `.env.local.example`에는 키 이름만 기록한다.
7. OpenAI API 호출은 서버 라우트에서만 수행한다.
8. `OPENAI_IMAGE_MODEL` 기본값은 `gpt-image-2`로 둔다.
9. Supabase service role key는 서버 전용 코드에서만 사용한다.
10. customer와 admin 권한을 명확히 분리한다.
11. admin API는 반드시 role 검사를 통과해야 한다.
12. Agent는 DB를 직접 수정하지 않는다.
13. Agent는 변경안을 제안하고 admin 승인 후에만 반영한다.
14. Agent 제안과 승인 이력은 저장해야 한다.
15. 결제는 v1에서 목업으로만 구현한다.
16. 실제 PG, 카드 승인, 영수증 발급은 구현하지 않는다.
17. 고객 사진은 공개 URL로 노출하지 않는다.
18. AI 착용 이미지 기능은 업로드 동의와 파일 검증을 포함한다.
19. 상품 이미지는 레이아웃 흔들림이 없도록 고정 비율을 사용한다.
20. UI는 미니멀 럭셔리 방향을 유지한다.
21. 과한 장식, 카드 중첩, 랜딩페이지만 있는 화면을 피한다.
22. 첫 화면은 실제 쇼핑 경험으로 시작한다.
23. 모바일과 데스크톱 모두 확인한다.
24. 새 기능에는 최소한의 테스트를 추가한다.
25. 권한, API, 결제 목업, AI 플로우는 우선 테스트 대상이다.
26. 구현 후 `npm run lint`와 `npm test`를 실행한다.
27. 프론트 주요 변경 후 브라우저로 화면을 확인한다.
28. 사용자 변경사항은 되돌리지 않는다.
29. 파괴적 git 명령은 명시 요청 없이 실행하지 않는다.
30. 커밋은 기능 단위로 작게 나눈다.
31. 구현 범위는 `docs/plan.md`의 마일스톤 우선순위를 따른다.
32. P0 작업을 끝내기 전 P2 작업을 먼저 확장하지 않는다.
